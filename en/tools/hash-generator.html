Frequently Asked Questions

What is a cryptographic hash?
A cryptographic hash is a fixed-size string generated from any input data. The same input always produces the same hash. Even a tiny change in input creates a completely different hash. Hashes are one-way — you cannot reverse them to get the original data.
SHA-256 vs SHA-512 — which should I use?
SHA-256 produces a 256-bit (64-character hex) hash and is the industry standard for most use cases including SSL certificates and Bitcoin. SHA-512 produces a 512-bit (128-character) hash, offering more resistance against brute force but with larger output.
Is MD5 still safe to use?
MD5 is NOT cryptographically secure for security-sensitive uses — it has known collision vulnerabilities. It's fine for checksums (verifying file integrity) but should not be used for password hashing or digital signatures. Use SHA-256 or SHA-512 for security purposes.
How do I verify a file's integrity with a hash?
Use the "Hash File" tab to generate the SHA-256 hash of the downloaded file. Compare it with the hash provided by the software publisher. If they match exactly, the file is authentic and unmodified.
