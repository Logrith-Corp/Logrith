Frequently Asked Questions

How secure are the generated passwords?
Very secure. The generator uses the Web Crypto API (crypto.getRandomValues) which is cryptographically secure randomness, the same standard used by security software and operating systems.
Are my passwords stored anywhere?
No. Passwords are generated purely in your browser using JavaScript. Nothing is sent to any server. The history shown is only in your browser's memory and disappears when you close the tab.
What makes a strong password?
A strong password is: at least 12 characters long (16+ is better), uses a mix of uppercase, lowercase, numbers and symbols, is not based on dictionary words, and is unique to each account.
What is the "Exclude Ambiguous" option?
It removes characters that look similar when typed: 0 (zero) vs O (letter), 1 (one) vs l (lowercase L) vs I (uppercase i). Useful for passwords that need to be read aloud or typed manually.
How do I use the bulk generator?
Enter the number of passwords you need (max 100), choose a separator, and click Generate Bulk. All passwords use the same settings from the main generator above.

Why You Need Strong Passwords

Data breaches expose millions of passwords every year. Using weak or reused passwords is one of the biggest security risks for individuals and businesses. A strong, unique password for each account — combined with a password manager — is the most effective protection.

Logrith's password generator uses cryptographically secure randomness (the Web Crypto API) to ensure no pattern can be predicted. Unlike online generators that might log your requests, this tool generates everything locally in your browser — your passwords never leave your device.
