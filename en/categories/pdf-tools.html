About Logrith PDF Tools

PDF manipulation typically requires expensive desktop software like Adobe Acrobat, or online services that upload your documents to their servers — raising serious privacy concerns for contracts, financial statements, and personal records. Logrith's PDF tools eliminate both problems by running entirely in your browser using PDF-Lib, a mature open-source JavaScript library that handles all PDF operations client-side with no server communication whatsoever.

The PDF Merge tool is the most commonly used — it combines multiple PDFs in any order you choose, with a drag-and-drop interface to rearrange files before merging. This is ideal for combining scanned pages, appending appendices to reports, or assembling multi-section documents. The PDF Split tool does the reverse, letting you extract specific pages using flexible range notation (e.g., "1-5, 8, 10-end").

For document creation from images, the JPG to PDF tool converts scanned documents, photos, and screenshots into properly formatted PDF files. It supports A4, US Letter, and fit-to-image page sizes. The complementary PDF to JPG tool renders each page as a raster image using PDF.js, with adjustable DPI scaling for different quality requirements — from 72 DPI for web thumbnails to 216 DPI for near-print quality.

Frequently Asked Questions

Is it safe to process my PDF documents here?
Yes, completely safe. Logrith uses PDF-Lib, which runs 100% in your browser's JavaScript environment. No PDF file is transmitted over the network — you can verify this in Chrome DevTools Network tab.
How do I merge PDF files in a specific order?
In the PDF Merge tool, upload all your PDFs, then drag the file rows to reorder them before clicking Merge. The final PDF will have pages in exactly the order shown in the file list.
Can I extract just certain pages from a PDF?
Yes. In the PDF Split tool, select "Extract Page Range" and enter something like "2-5, 8, 12-15" to extract those specific pages into a new PDF document.
Why does PDF to JPG require an internet connection?
PDF.js (used for rendering PDF pages as images) loads from a CDN on first use. Once loaded, the actual conversion is local. PDF-Lib (used for merge, split, rotate) also loads from CDN. After initial page load, you can use most tools offline.
What quality is the PDF to JPG conversion?
You can choose from Standard (72 DPI), Medium (108 DPI), High (144 DPI), or Ultra (216 DPI). For screen use, High is recommended. For printing or archiving, Ultra gives near-print quality at larger file sizes.
Can I convert a Word document to PDF?
Not directly through browser JavaScript — Word (.docx) to PDF conversion requires server processing. However, you can use Microsoft Word's "Print to PDF" or "Save as PDF" option in any browser or Office application, then use Logrith's other PDF tools on the resulting file.
What's the maximum PDF file size?
Each PDF file has a 20MB limit. Very large PDFs (100+ page reports, scanned books) may be slow to process on lower-end devices due to browser memory constraints. Splitting oversized PDFs before merging can help performance.
Will merging PDFs reduce quality?
No. PDF-Lib copies page objects directly from source PDFs — no re-encoding or re-rendering occurs. The merged PDF is bit-for-bit identical to the source pages at their original quality.
Can I rotate just one page of a PDF?
Yes. In the PDF Rotate tool, select "Specific Page Range" and enter just the page number you want to rotate (e.g., "3" for page 3). Only that page will be rotated in the output PDF.
How do I convert scanned images into a searchable PDF?
Logrith's JPG to PDF creates image-based PDFs — the text in scanned images is not machine-readable. For OCR (optical character recognition) to make PDFs searchable, you currently need tools like Adobe Acrobat, ABBYY FineReader, or Google Drive's built-in OCR. This is on Logrith's roadmap.
