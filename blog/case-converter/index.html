Common Case Formats Explained

UPPERCASE

All letters are converted to capital letters. Example: "HELLO WORLD". Commonly used for headings, titles, acronyms, and emphasis in both formal and informal writing. In code, often used for constants.

lowercase

All letters are in lowercase. Example: "hello world". Standard format for body text and default format in many applications. Most web content uses lowercase for better readability.

Title Case

First letter of each word is capitalized. Example: "Hello World". Commonly used for titles, headings, article headlines, and proper nouns. Essential for professional documents and publications.

Sentence case

Only the first letter of the sentence is capitalized, rest in lowercase. Example: "Hello world". Standard format for regular text paragraphs and formal writing. Most natural and readable for body text.

camelCase

First word lowercase, subsequent words capitalized without spaces. Example: "helloWorld". Common in programming for variable and function names. Default convention in JavaScript, Java, and many modern languages.

PascalCase

All words capitalized without spaces, including the first word. Example: "HelloWorld". Used in programming for class names, type names, and namespace identifiers. Standard in C#, TypeScript, and Pascal.

snake_case

Words separated by underscores, all lowercase. Example: "hello_world". Common in Python, database naming, file naming, and URL parameters. Provides clear word separation without spaces.

kebab-case

Words separated by hyphens, all lowercase. Example: "hello-world". Used in URLs, CSS classes, HTML attributes, and file names. Provides readable separation while remaining web-safe.

How Different Languages Use Case Conventions

JavaScript and TypeScript

JavaScript developers predominantly use camelCase for variables and functions, and PascalCase for class names and constructors.

const firstName = "John"; // Variable function getUserData() {} // Function class UserManager {} // Class

Python

Python convention (PEP 8) specifies snake_case for variables and functions, UPPERCASE for constants, and PascalCase for class names.

user_name = "John" # Variable def get_user_data(): pass # Function class UserManager: pass # Class API_KEY = "abc123" # Constant

C# and .NET

C# follows PascalCase for most identifiers including properties, methods, and classes, with camelCase used internally.

public string FirstName { get; set; } // Property public void GetUserData() {} // Method public class UserManager {} // Class

Database Naming

Databases typically use snake_case for table and column names for consistency and SQL compatibility.

CREATE TABLE users ( user_id INT, first_name VARCHAR(50), last_name VARCHAR(50), email_address VARCHAR(100) );

How do I know which case format to use?
The case format depends on your context. For writing, use Title Case for titles and Sentence case for paragraphs. Yes, the case converter is helpful for refactoring variable names when moving code between languages or updating naming conventions. However, for large-scale code refactoring, IDE built-in refactoring tools provide more comprehensive solutions with better context awareness.
